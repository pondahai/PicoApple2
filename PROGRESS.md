# 📝 Pico Apple II Emulator 開發進度筆記

## 1. 專案狀態更新 (2026-03-24) - [重大里程碑：磁碟寫入完美修復]
- **[核心技術修復 1] 實施「Q7 寫入模式鎖定 (Q7 Write Mode Lock)」**。
    - **發現**：修正了 `disk2.rs` 中的一個隱藏 Bug。原先邏輯誤判「只有 Q6 && Q7 同時開啟」才寫入，導致在 Q6 切換時模擬器會短暫跳回「讀取模式」，使 `data_latch` 被舊資料覆蓋。
    - **修正**：只要 Q7 開啟，強制鎖定為寫入模式並停用讀取電路。這確保了軟體輸出的位元流能 100% 完整地「刻入」磁軌緩衝區。
- **[核心技術修復 2] 實施「具備髒位元意識的解碼 (Dirty-aware Denibblization)」**。
    - **背景**：Pico 因 RAM 限制採用「單軌分頁」，磁軌緩衝區會同時存在「初始化生成的舊資料」與「軟體寫入的新資料」。
    - **修正**：利用 `dirty_mask` 紀錄被軟體改動的位元組。解碼器（Denibblizer）現在會掃描整條磁軌，並優先提取 `dirty_count` 最高的扇區版本。
    - **成果**：**徹底解決了 `SAVE` 後內容未變的問題**。現在 `SAVE TEST` 後執行 `CATALOG` 可正確看到新檔案。
- **[物理層驗證]** 磁碟資料持久化已 100% 穩定。
    - 現象：`SAVE` 指令後，終端機顯示 `SD: Write & Physical Verify [SUCCESS]`。
    - 驗證：Hex Dump 顯示磁軌目錄（Track 17）的內容已隨檔案儲存正確更新。
